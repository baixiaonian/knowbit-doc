句子2: "你好世界"
</code></pre><p><strong>问题</strong>：神经网络无法直接处理文本字符串，必须先转换为数字。</p><h3 id="heading-540">0.2 词汇表 (Vocabulary)</h3><p>词汇表是所有可能 token 的集合，每个 token 分配一个唯一的 ID。</p><pre class="code-block"><code class="language-plaintext">词汇表 (Vocabulary):

┌────────┬──────────┬─────────────────────────┐

├────────┼──────────┼─────────────────────────┤
│   │    0     │ 填充符，用于对齐序列长度  │
│   │    1     │ 未知词，不在词汇表中      │
│ 我     │    2     │ 常用字                   │
│ 喜欢   │    3     │ 常用词（词粒度）          │
│ 学习   │    4     │ 常用词                   │
│ AI     │    5     │ 专有名词                 │
│ 你     │    6     │ 常用字                   │
│ 好     │    7     │ 常用字                   │
│ 世界   │    8     │ 常用词                   │
│ 了     │    9     │ 助词                     │
└────────┴──────────┴─────────────────────────┘

词汇表大小 vocab_size = 10
</code></pre><h3 id="heading-1299">0.3 分词方式对比</h3><p>实际应用中有多种分词方式：</p><pre class="code-block"><code class="language-plaintext">┌────────────────────────────────────────────────────────────────────────┐
│                        三种主流分词方式                                 │
├────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 词粒度分词 (Word-level)                                             │
│     "我喜欢学习AI" → ["我", "喜欢", "学习", "AI"]                        │
│     优点：语义完整                                                       │
│     缺点：词汇表大，未登录词(OOV)问题严重                                │
│                                                                         │
│  2. 字粒度分词 (Character-level)                                        │
│     "我喜欢学习AI" → ["我", "喜", "欢", "学", "习", "A", "I"]            │
│     优点：词汇表小，无OOV问题                                            │
│     缺点：序列长，语义不完整                                             │
│                                                                         │
│  3. 子词分词 (Subword-level) ← 主流方案                                 │
│     BPE/WordPiece/SentencePiece                                         │
│     "我喜欢学习AI" → ["我", "喜欢", "学习", "AI"]                        │
│     优点：平衡词汇表大小和序列长度，能有效处理未登录词                    │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
</code></pre><h3 id="heading-2727">0.4 分词过程详解</h3><p>以 <strong>词粒度分词</strong> 为例，逐步展示转换过程：</p><h4 id="heading-2761">Step 1: 文本清洗</h4><pre class="code-block"><code class="language-plaintext">原始文本: "我喜欢学习AI"

</code></pre><h4 id="heading-2887">Step 2: 分词 (Tokenization)</h4><pre class="code-block"><code class="language-plaintext">句子1: "我喜欢学习AI"

       ["我", "喜欢", "学习", "AI"]

        ↓ 分词器
       ["你", "好", "世界"]
</code></pre><p><strong>分词过程详解</strong>（以句子1为例）：</p><pre class="code-block"><code class="language-plaintext">输入字符串: "我喜欢学习AI"
             │
             ▼
┌─────────────────────────────────────────────────────────────────┐

│                                                                  │

│         ├─ 查词汇表: "我" 存在 (ID=2)                            │
│         ├─ 尝试更长: "我喜欢" 不存在                             │
│         └─ 切分结果: "我"                                        │
│                                                                  │
│ 位置 1: 当前字符 "喜"                                            │
│         ├─ 查词汇表: "喜" 不存在作为独立词                        │
│         ├─ 尝试更长: "喜欢" 存在 (ID=3)                          │
│         ├─ 尝试更长: "喜欢学" 不存在                             │
│         └─ 切分结果: "喜欢"                                      │
│                                                                  │
│ 位置 3: 当前字符 "学"                                            │
│         ├─ 查词汇表: "学" 不存在作为独立词                        │
│         ├─ 尝试更长: "学习" 存在 (ID=4)                          │
│         ├─ 尝试更长: "学习A" 不存在                              │
│         └─ 切分结果: "学习"                                      │
│                                                                  │
│ 位置 5: 当前字符 "A"                                             │
│         ├─ 查词汇表: "A" 不存在                                  │
│         ├─ 尝试更长: "AI" 存在 (ID=5)                            │
│         └─ 切分结果: "AI"                                        │

</code></pre><h4 id="heading-4867">Step 3: Token → ID 映射</h4><pre class="code-block"><code class="language-plaintext">句子1 分词结果: ["我", "喜欢", "学习", "AI"]
                   │     │      │     │

                   │     │      │     │

                   │     │     │
                   ▼     ▼     ▼

</code></pre><p><strong>映射过程详解</strong>：</p><pre class="code-block"><code class="language-plaintext">┌───────────────────────────────────────────────────────────────┐
│                    Token → ID 映射表                          │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│   Token 字符串        查表操作              Token ID          │
│   ────────────        ────────              ────────          │
│                                                               │
│   "我"         →    vocab["我"]    →       2                 │
│   "喜欢"       →    vocab["喜欢"]  →       3                 │
│   "学习"       →    vocab["学习"]  →       4                 │

│                                                               │

</code></pre><h4 id="heading-6417">Step 4: 序列对齐 (Padding/Truncation)</h4><pre class="code-block"><code class="language-plaintext">当前 Token IDs:
句子1: [2, 3, 4, 5]  长度=4
句子2: [6, 7, 8]     长度=3

问题: 一个 batch 内的序列长度必须一致！
      神经网络要求输入是规整的张量。

解决方案: Padding (填充) 或 Truncation (截断)

假设目标长度 seq_len = 4:

句子1: [2, 3, 4, 5]  ✓ 长度刚好，无需处理
句子2: [6, 7, 8, ?]  ✗ 长度不足，需要填充
              │
              ▼
句子2: [6, 7, 8, 0]  ← 用 0 () 填充
              ↑
           用  填充到目标长度
</code></pre><p><strong>Padding 策略详解</strong>：</p><pre class="code-block"><code class="language-plaintext">┌───────────────────────────────────────────────────────────────┐
│                     Padding 策略                              │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  1. 右填充 (Post-padding) ← 常用                              │
│     [6, 7, 8] → [6, 7, 8, 0, 0]                              │
│                   ↑↑↑  ↑  ↑                                  │
│                  真实  填充                                    │
│                                                               │
│  2. 左填充 (Pre-padding)                                      │
│     [6, 7, 8] → [0, 0, 6, 7, 8]                              │
│                   ↑  ↑  ↑↑↑                                  │
│                  填充  真实                                    │
│                                                               │
│  3. 截断 (Truncation) - 当序列超过最大长度时                  │
│     [1,2,3,4,5,6,7,8] (长度8, 最大长度4)                     │
│     → [1, 2, 3, 4]  保留前4个                                │
│     或 [5, 6, 7, 8]  保留后4个                                │
│                                                               │
└───────────────────────────────────────────────────────────────┘
</code></pre><h4 id="heading-8085">Step 5: 添加特殊 Token (可选)</h4><pre class="code-block"><code class="language-plaintext">实际应用中，通常会添加特殊 token:

句子1: [2, 3, 4, 5]  →  [, 2, 3, 4, 5, ]
                         ↑                   ↑
                      开始符              结束符

其中:
-  (Start of Sequence): 序列开始标记，告诉模型"开始生成"
-  (End of Sequence): 序列结束标记，告诉模型"生成结束"
-  (Classification): 用于分类任务，代表整个序列的表示
-  (Separator): 用于分隔两个句子

本示例简化处理，不添加这些特殊 token。
</code></pre><h4 id="heading-8467">Step 6: 创建 Attention Mask</h4><pre class="code-block"><code class="language-plaintext">为什么需要 Attention Mask？

在计算注意力时，padding 位置不应该被关注（它们没有实际意义）。
因此需要创建一个 mask 来屏蔽这些位置。

句子1: [2, 3, 4, 5]  →  mask: [1, 1, 1, 1]  全部是真实 token
句子2: [6, 7, 8, 0]  →  mask: [1, 1, 1, 0]  最后一个是 padding

Attention Mask:
形状: [batch_size, seq_len] = [2, 4]

[[1, 1, 1, 1],
 [1, 1, 1, 0]]

其中:
  1 = 真实 token，参与注意力计算
  0 = padding，在注意力计算中被屏蔽
</code></pre><h3 id="heading-8825">0.5 最终输入数据</h3><pre class="code-block"><code class="language-plaintext">┌─────────────────────────────────────────────────────────────────┐

</code></pre><h3 id="heading-10092">0.6 未登录词 (OOV) 处理</h3><pre class="code-block"><code class="language-plaintext">如果遇到词汇表中没有的词怎么办？

示例: "我喜欢编程"
分词: ["我", "喜欢", "编程"]
                     ↑
              "编程" 不在词汇表中！

处理方式:

┌───────────────────────────────────────────────────────────────┐
│ 1. 使用  token (ID=1):                                   │
│    ["我", "喜欢", "编程"] → [2, 3, 1]                         │
│                                ↑                               │
│                                                           │
│    缺点: 丢失了语义信息                                        │
├───────────────────────────────────────────────────────────────┤
│ 2. 子词分词 (Subword):                                        │
│    "编程" → ["编", "程"] (如果这两个字在词汇表中)              │
│    最终: ["我", "喜欢", "编", "程"] → [2, 3, ?, ?]            │
│    优点: 保留更多信息                                          │
├───────────────────────────────────────────────────────────────┤
│ 3. BPE (Byte Pair Encoding):                                  │
│    词汇表: ["编", "程", "编程"] (如果有)                       │
│    "编程" → ["编程"] (如果"编程"在词汇表)                     │
│          或 ["编", "程"] (如果分开存在)                       │
│          或 ["编", "程"] (按字节级拆分)                       │
├───────────────────────────────────────────────────────────────┤
│ 4. WordPiece:                                                 │
│    "编程" → ["编", "##程"]                                    │
│                 ↑                                             │
│            ## 表示这是后缀，不是独立词                         │

句子2: [6, 7, 8, 0]  # "你好世界" + padding
</code></pre><h3 id="heading-15987">嵌入矩阵</h3><p>嵌入矩阵是一个查找表，每行代表一个词的嵌入向量。这是一个可学习的参数矩阵：</p><pre class="code-block"><code class="language-plaintext">嵌入矩阵 E:
形状: [vocab_size, d_model] = [10, 4]

        列0    列1    列2    列3
      ┌──────┬──────┬──────┬──────┐
ID=0  │ 0.00 │ 0.10 │ 0.20 │ 0.30 │  ←  的嵌入
      ├──────┼──────┼──────┼──────┤
ID=1  │ 0.40 │ 0.50 │ 0.60 │ 0.70 │  ←  的嵌入
      ├──────┼──────┼──────┼──────┤
ID=2  │ 0.80 │ 0.90 │ 1.00 │ 1.10 │  ← "我" 的嵌入
      ├──────┼──────┼──────┼──────┤

      ├──────┼──────┼──────┼──────┤

      ├──────┼──────┼──────┼──────┤
ID=5  │ 2.00 │ 2.10 │ 2.20 │ 2.30 │  ← "AI" 的嵌入
      ├──────┼──────┼──────┼──────┤
ID=6  │ 2.40 │ 2.50 │ 2.60 │ 2.70 │  ← "你" 的嵌入
      ├──────┼──────┼──────┼──────┤

- 输入 [A, B, C] 和输入 [C, B, A] 计算结果完全一样（除了位置）
- 因为 Q@K^T 只看内容，不看位置
- 这意味着模型无法区分 "我喜欢你" 和 "你喜欢我"

解决方案: 给每个位置加一个独特的位置向量
</code></pre><h3 id="heading-18613">位置编码矩阵</h3><p>使用正弦/余弦函数生成（这种编码方式可以泛化到任意长度）：</p><pre class="code-block"><code class="language-plaintext">位置编码 PE:
形状: [seq_len, d_model] = [4, 4]

计算公式:
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

其中:
- pos 是位置索引 (0, 1, 2, ...)
- i 是维度索引 (0, 1, ...)
- 2i 和 2i+1 分别是偶数和奇数维度

具体计算:
位置0:
  维度0: sin(0/10000^0) = sin(0) = 0.0000
  维度1: cos(0/10000^0) = cos(0) = 1.0000
  维度2: sin(0/10000^0.5) = sin(0) = 0.0000
  维度3: cos(0/10000^0.5) = cos(0) = 1.0000
  → [0.0000, 1.0000, 0.0000, 1.0000]

位置1:
  维度0: sin(1/1) = sin(1) = 0.8415
  维度1: cos(1/1) = cos(1) = 0.5403
  维度2: sin(1/100) = sin(0.01) = 0.0100
  维度3: cos(1/100) = cos(0.01) = 1.0000
  → [0.8415, 0.5403, 0.0100, 1.0000]

位置2:
  → [0.9093, -0.4161, 0.0200, 0.9998]

位置3:
  → [0.1411, -0.9900, 0.0300, 0.9996]

位置编码矩阵:
        列0     列1     列2     列3
      ┌───────┬───────┬───────┬───────┐
位置0 │ 0.0000│ 1.0000│ 0.0000│ 1.0000│
      ├───────┼───────┼───────┼───────┤
位置1 │ 0.8415│ 0.5403│ 0.0100│ 1.0000│
      ├───────┼───────┼───────┼───────┤
位置2 │ 0.9093│-0.4161│ 0.0200│ 0.9998│
      ├───────┼───────┼───────┼───────┤
位置3 │ 0.1411│-0.9900│ 0.0300│ 0.9996│
      └───────┴───────┴───────┴───────┘
</code></pre><h3 id="heading-19754">计算过程</h3><pre class="code-block"><code class="language-plaintext">公式: output = embedding + positional_encoding

</code></pre><h3 id="heading-20288">输出</h3><pre class="code-block"><code class="language-plaintext">位置编码后输出:

</code></pre><hr><h2 id="heading-20820">第三步：自注意力计算 (Self-Attention)</h2><p><strong>核心公式</strong>：<code>Attention(Q,K,V) = softmax(QK^T / sqrt(d_k)) × V</code></p><h3 id="heading-20904">3.1 计算 Q, K, V</h3><p>通过线性变换将输入投影到三个不同的空间：</p><pre class="code-block"><code class="language-plaintext">输入 x (位置编码后的输出):
形状: [batch_size, seq_len, d_model] = [2, 4, 4]

权重矩阵 (假设值):
W_q: [d_model, d_model] = [4, 4]  # Query 投影
W_k: [d_model, d_model] = [4, 4]  # Key 投影
W_v: [d_model, d_model] = [4, 4]  # Value 投影
</code></pre><p>计算过程（以句子1位置0为例）：</p><pre class="code-block"><code class="language-plaintext">x[0][0] = [0.8000, 1.9000, 1.0000, 2.1000]  # "我" 的位置编码后向量

Q[0][0] = x[0][0] @ W_q
        = [0.8000, 1.9000, 1.0000, 2.1000] @ [[w11, w12, w13, w14],
                                               [w21, w22, w23, w24],
                                               [w31, w32, w33, w34],
                                               [w41, w42, w43, w44]]
        
假设简化计算后:
Q[0][0] = [0.12, -0.08, 0.25, 0.15]  # "我" 的 Query 向量

K[0][0] = x[0][0] @ W_k = [0.35, 0.22, -0.10, 0.40]  # "我" 的 Key 向量

V[0][0] = x[0][0] @ W_v = [0.18, 0.30, 0.05, -0.12]  # "我" 的 Value 向量
</code></pre><h3 id="heading-21745">Q, K, V 的含义</h3><pre class="code-block"><code class="language-plaintext">┌─────────────────────────────────────────────────────────────────┐
│                    Q, K, V 的直观理解                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Query (Q): "我要查询什么？"                                     │
│    - 代表当前位置想要获取的信息类型                               │
│    - 例如 "我" 这个位置的 Q 可能表示"我想知道有没有表示情感的内容" │
│                                                                  │
│  Key (K): "我是什么？"                                           │
│    - 代表当前位置可以提供的信息特征                               │
│    - 用于与其他位置的 Q 匹配                                     │
│    - 例如 "喜欢" 的 K 可能表示"我是一个表示情感的词"              │
│                                                                  │
│  Value (V): "我的内容是什么？"                                   │
│    - 代表当前位置实际传递的信息内容                               │
│    - 用于被注意力加权后传递给其他位置                             │
│                                                                  │
│  匹配过程:                                                       │
│    Q(我) · K(喜欢) = 高分 → "我" 关注 "喜欢"                     │
│    Q(我) · K(AI) = 低分 → "我" 不太关注 "AI"                     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
</code></pre><h3 id="heading-23119">Q, K, V 输出</h3><pre class="code-block"><code class="language-plaintext">Q (Query):
形状: [batch_size, seq_len, d_model] = [2, 4, 4]

  位置0 "我":   [0.12, -0.08, 0.25, 0.15]

  位置2 "学习": [0.67, -0.15, 0.42, 0.55]
  位置3 "AI":   [0.23, 0.18, 0.35, -0.12]

形状: [batch_size, seq_len, d_model] = [2, 4, 4]

句子1:
  位置0 "我":   [0.35, 0.22, -0.10, 0.40]
  位置1 "喜欢": [0.28, 0.45, 0.15, -0.20]
  位置2 "学习": [0.50, 0.18, 0.30, 0.25]
  位置3 "AI":   [0.15, 0.38, -0.05, 0.42]

V (Value):
形状: [batch_size, seq_len, d_model] = [2, 4, 4]

句子1:
  位置0 "我":   [0.18, 0.30, 0.05, -0.12]
  位置1 "喜欢": [0.42, 0.15, 0.28, 0.35]
  位置2 "学习": [0.25, 0.48, -0.08, 0.22]
  位置3 "AI":   [0.35, 0.22, 0.15, 0.18]
</code></pre><h3 id="heading-23787">3.2 计算注意力分数</h3><pre class="code-block"><code class="language-plaintext">公式: scores = Q @ K^T / sqrt(d_k)

其中 d_k = d_model = 4, sqrt(4) = 2
</code></pre><p>计算 Q @ K^T（点积）：</p><pre class="code-block"><code class="language-plaintext">K^T (K的转置):
形状: [d_model, seq_len] = [4, 4]

Q @ K^T 计算:
形状: [seq_len, seq_len] = [4, 4]

位置0 对所有位置的关注分数:
  score[0][0] = Q[0]·K[0] 
              = 0.12×0.35 + (-0.08)×0.22 + 0.25×(-0.10) + 0.15×0.40
              = 0.0420 - 0.0176 - 0.0250 + 0.0600
              = 0.0594

  score[0][1] = Q[0]·K[1]
              = 0.12×0.28 + (-0.08)×0.45 + 0.25×0.15 + 0.15×(-0.20)
              = 0.0336 - 0.0360 + 0.0375 - 0.0300
              = 0.0051

  score[0][2] = Q[0]·K[2]
              = 0.12×0.50 + (-0.08)×0.18 + 0.25×0.30 + 0.15×0.25
              = 0.0600 - 0.0144 + 0.0750 + 0.0375
              = 0.1581

  score[0][3] = Q[0]·K[3]
              = 0.12×0.15 + (-0.08)×0.38 + 0.25×(-0.05) + 0.15×0.42
              = 0.0180 - 0.0304 - 0.0125 + 0.0630
              = 0.0381
</code></pre><h3 id="heading-24664">缩放后的分数</h3><pre class="code-block"><code class="language-plaintext">scores = Q @ K^T / sqrt(d_k) = Q @ K^T / 2

形状: [batch_size, seq_len, seq_len] = [2, 4, 4]

句子1:
            "我"    "喜欢"  "学习"   "AI"    ← 被查询的位置
            位置0   位置1   位置2   位置3
         ┌────────┬────────┬────────┬────────┐

         ├────────┼────────┼────────┼────────┤

         ├────────┼────────┼────────┼────────┤

         └────────┴────────┴────────┴────────┘
</code></pre><p><strong>解读</strong>：分数矩阵中 <code>scores[i][j]</code> 表示位置 i 对位置 j 的原始关注程度。</p><ul><li><p>分数越高，表示关注度越高</p></li><li><p>例如：位置2("学习") 对位置2("学习") 的分数最高(0.1462)，说明"学习"最关注自己</p></li></ul><h3 id="heading-25387">3.3 Softmax 归一化</h3><pre class="code-block"><code class="language-plaintext">公式: attention_weights = softmax(scores, dim=-1)

            位置0   位置1   位置2   位置3

作用: 
1. 缓解梯度消失 - 梯度可以直接通过残差路径传递
2. 允许模型学习"恒等映射" - 如果子层没用，可以学成 0
3. 使深层网络更容易训练
</code></pre><pre class="code-block"><code class="language-plaintext">示例（自注意力后的残差）:

输入 x (注意力前):
句子1位置0: [0.19, 0.03, 0.18, 0.09]

注意力输出:
句子1位置0: [0.15, 0.08, 0.22, 0.05]

残差连接后:
句子1位置0: [0.19+0.15, 0.03+0.08, 0.18+0.22, 0.09+0.05]
          = [0.34, 0.11, 0.40, 0.14]
</code></pre><h3 id="heading-32120">层归一化 (Layer Normalization)</h3><pre class="code-block"><code class="language-plaintext">公式: LayerNorm(x) = (x - μ) / sqrt(σ² + ε) × γ + β

对单个样本的所有特征进行归一化
</code></pre><pre class="code-block"><code class="language-plaintext">示例计算（位置0）:

输入: [0.34, 0.11, 0.40, 0.14]

Step 1: 计算均值
μ = (0.34 + 0.11 + 0.40 + 0.14) / 4 = 0.2475

Step 2: 计算方差
σ² = [(0.34-0.2475)² + (0.11-0.2475)² + (0.40-0.2475)² + (0.14-0.2475)²] / 4
   = [0.0086 + 0.0189 + 0.0233 + 0.0116] / 4
   = 0.0156

Step 3: 归一化
ε = 1e-5 (防止除零)
normalized = (x - μ) / sqrt(σ² + ε)
           = [0.34-0.2475, 0.11-0.2475, 0.40-0.2475, 0.14-0.2475] / sqrt(0.0156)
           = [0.0925, -0.1375, 0.1525, -0.1075] / 0.1249
           = [0.7406, -1.1010, 1.2210, -0.8606]

Step 4: 缩放和平移 (γ, β 是可学习参数，假设 γ=1, β=0)
output = normalized × γ + β = [0.74, -1.10, 1.22, -0.86]
</code></pre><h3 id="heading-32816">为什么用 LayerNorm 而不是 BatchNorm？</h3><pre class="code-block"><code class="language-plaintext">┌─────────────────────────────────────────────────────────────────┐
│           BatchNorm vs LayerNorm                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  BatchNorm: 对 batch 中所有样本的同一特征归一化                  │
│                                                                  │
│    样本1: [x1, x2, x3, x4]                                      │
│    样本2: [y1, y2, y3, y4]                                      │
│    样本3: [z1, z2, z3, z4]                                      │
│           ↓  ↓   ↓   ↓                                          │
│         归一化每一列                                             │
│                                                                  │
│  LayerNorm: 对单个样本的所有特征归一化                          │
│                                                                  │
│    样本1: [x1, x2, x3, x4]  → 归一化                            │
│    样本2: [y1, y2, y3, y4]  → 归一化                            │
│    样本3: [z1, z2, z3, z4]  → 归一化                            │
│                                                                  │
│  Transformer 使用 LayerNorm 因为:                               │
│    1. 不依赖 batch size                                         │
│    2. 序列长度可变时也能正常工作                                │
│    3. 每个 token 独立归一化，适合序列任务                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
</code></pre><hr><h2 id="heading-34402">第七步：完整 Encoder 层数据流</h2><pre class="code-block"><code class="language-plaintext">┌─────────────────────────────────────────────────────────────────┐

的方阵，表示位置间的关注关系</p></li><li><p>最终输出形状变为词汇表大小，用于预测下一个词</p></li></ul><hr><h2 id="heading-45079">关键计算公式汇总</h2><pre class="code-block"><code class="language-plaintext">1. Tokenization:
   tokens = tokenize(text)             # 字符串 → token 列表
   ids = [vocab[t] for t in tokens]    # token → ID

2. 词嵌入:
   x = Embedding(token_ids)            # [B, L] → [B, L, d]

3. 位置编码:
   x = x + PE(position)                # 加上位置向量

4. 自注意力:
   Q = x @ W_q                         # 查询向量
   K = x @ W_k                         # 键向量
   V = x @ W_v                         # 值向量
   scores = Q @ K^T / √d_k             # 注意力分数
   attention = softmax(scores)         # 归一化
   output = attention @ V              # 加权求和

5. 多头注意力:
   heads = split(Q,K,V, num_heads)     # 分割成多头
   outputs = [Attention(h) for h in heads]  # 每头独立计算
   output = concat(outputs) @ W_o      # 拼接并投影

6. 前馈网络:
   FFN(x) = ReLU(x @ W_1) @ W_2        # 扩展→激活→压缩

7. 残差 + LayerNorm:
   output = LayerNorm(x + Sublayer(x)) # 残差连接后归一化

8. 输出:
   logits = output @ W_out             # 投影到词汇表
   probs = softmax(logits)             # 概率分布
   prediction = argmax(probs)          # 预测 token
</code></pre><hr><h2 id="heading-46068">总结</h2><p>Transformer 的核心是<strong>自注意力机制</strong>，它让每个位置都能直接与所有其他位置交互，捕捉长距离依赖关系。</p><p><strong>数据流特点</strong>：</p><ol><li><p>维度稳定：<code>[batch, seq_len, d_model]</code> 贯穿全程</p></li><li><p>信息融合：每个位置通过注意力聚合全局信息</p></li><li><p>残差学习：便于训练深层网络</p></li><li><p>并行计算：所有位置同时处理（训练时）</p></li></ol><p><strong>Tokenization 是被忽略但至关重要的一步</strong>：</p><ul><li><p>原始文本 → 分词 → ID 映射 → Padding → 最终输入张量</p></li><li><p>不同的分词策略会显著影响模型性能</p></li></ul>
