<h1 id="_1">文档向量更新策略</h1><h2 id="_2">🎯 问题</h2><p>用户编辑文档时，前端每秒都会发起保存请求。 如何智能地更新向量，平衡成本、性能和实时性？</p><hr><h2 id="_3">💡 推荐方案：多层策略</h2><h3 id="1debounce">策略1：防抖机制（Debounce）⭐ 推荐</h3><p><strong>原理</strong>：只在用户停止编辑一段时间后才向量化</p><pre class="code-block"><code class="language-plaintext">用户编辑: t0 → t1 → t2 → t3 → t4 → (停止) → t4+3分钟

好处：
✅ 避免频繁调用API
✅ 降低成本（只在真正完成编辑后处理）
✅ 用户无感知（后台异步）
</code></pre><p><strong>实现</strong>：</p><ol><li><p>文档每次保存时： ```python # 只更新documents表 document.content = new_content document.updated_at = now()</p></li></ol><p># 标记需要重新向量化（不立即执行） if content_changed: document.vectorization_status = 'pending' ```</p><ol><li><p>后台定时任务（每分钟运行）： ```python # 查找：状态=pending 且 3分钟内无更新的文档 docs = SELECT * FROM documents WHERE vectorization_status = 'pending' AND updated_at &lt; (now() - interval '3 minutes')</p></li></ol><p># 异步向量化这些文档 for doc in docs: vectorize_in_background(doc) ```</p><h3 id="2">策略2：内容哈希检测</h3><p><strong>原理</strong>：只在内容真正改变时才重新向量化</p><pre class="code-block"><code class="language-plaintext"># 计算内容哈希old_hash = MD5(old_content)new_hash = MD5(new_content)

if old_hash != new_hash:
    # 内容真正改变了
    mark_for_vectorization()else:
    # 只是标题、标签等元数据改变，不需要重新向量化
    skip_vectorization()</code></pre><p><strong>好处</strong>： - ✅ 避免不必要的向量化 - ✅ 节省成本 - ✅ 提高效率</p><h3 id="3">策略3：智能队列优先级</h3><p><strong>优先级排序</strong>：</p><pre class="code-block"><code class="language-plaintext">SELECT * FROM documentsWHERE vectorization_status = 'pending'ORDER BY    CASE         WHEN is_public = true THEN 1  -- 公开文档优先        ELSE 2    END,    updated_at DESC                   -- 最近更新的优先LIMIT 10</code></pre><hr><h2 id="_4">🏗️ 完整架构</h2><pre class="code-block"><code class="language-plaintext">┌─────────────────────────────────────────────┐
